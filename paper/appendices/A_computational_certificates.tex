\section{Computational Certificates and Reproducibility}\label{app:certs}

All paths in this appendix are relative to the companion repository root.  The scripts are deterministic exact-arithmetic checks:
they use integer or rational arithmetic and have no floating-point acceptance criterion.  Artifact-producing scripts are rerun as
\[
  \texttt{python -B <script> --write-json <json>}.
\]
When a script exposes Markdown output, the replay command also supplies \texttt{--write-md <md>}.  Report-only commands omit output flags when the script does not expose them.  The finite rows in these reports are edge checks or diagnostics; the all-$m$ claims are carried by polynomial
identities, integration-by-parts certificates, product/$\Phi$ block accounting, and ratio induction.

For readability, the internal replay IDs have the following public roles: S11A-H is the odd reference true-scalar link;
S11A-K local/global and S11A-transfer are the odd pair-standard true-scalar link; S8 and S9C are the holonomic
recurrence/positivity certificates for the two residual sequences; S12A is the signed-poset separator certificate; and
S12B is the translate-tiler divisibility certificate.  The filenames are stable repository identifiers, not additional
mathematical claims.

The rank theorem uses only the scoped two-standard-channel conclusion.  The S12 rows add only the separately stated
positive-only separator theorem for $X_n=\BtildeI_n(1,1)$, $n\ge4$.  The certificate table does not add a Type-D theorem,
a full $B_n$ fingerprint theorem, a classification theorem, a general Weyl theorem, or an enumeration novelty claim.
